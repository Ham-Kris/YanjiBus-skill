# 延吉公交实时查询

## 核心规则

1. 运行脚本：`bash ./yanji-bus.sh <线路号> [子线路号] [--from <站名> --to <站名>]`
2. 有出发地和目的地时必须传 `--from` 和 `--to`，脚本自动判断上下行方向
3. 只知道线路号时，子线路默认为 1；3路有 3 条子线路（1/2/3），需确认
4. 回复用户时，聚焦于**哪辆车还没到上车站、还有几站**，而不是罗列所有车辆

## Gotchas

- **已过站 ≠ 快到了**：车辆在第 N 站，用户上车站是第 M 站。下行方向中 N > M 意味着该车**已过站不可乘**；N < M 才是"还有 M−N 站"。绝对不能把"离目的地近"误判为"对用户有用"。
- **判断依据是上车站，不是下车站**：用户关心的是"车还没到我这里"，而不是"车离我目的地近"。
- **上行/下行是独立站序**：两套方向的站序编号独立，不可跨方向比较站号。
- **站名用模糊匹配**：`--from`/`--to` 传部分站名即可，如"中国银行"可匹配"中国银行延边分行"。
- **3路必须确认子线路**：3路共 3 条子线路终点不同，用户未指定时询问或默认子线路 1。
